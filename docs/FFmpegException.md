# FFmpegException

An exception thrown by the `ffmpeg-fluent` library when an FFmpeg process exits with a non-zero status code, providing details about the failure including the exit code, standard error output, and the executed command line.

## API

### `public int ExitCode`
The exit code returned by the FFmpeg process. This value is typically non-zero when an error occurs. Use this to determine the nature of the failure (e.g., invalid arguments, encoding error, or resource constraints).

### `public string StdErr`
The standard error output (`stderr`) captured from the FFmpeg process. Contains diagnostic messages, warnings, or errors generated by FFmpeg during execution. This string may be empty if no output was produced.

### `public string CommandLine`
The full command line used to invoke the FFmpeg process. Includes all arguments and options passed to FFmpeg. This can be used for debugging or reproducing the failure.

### `public FFmpegException(string message, int exitCode, string stdErr, string commandLine)`
Constructs a new `FFmpegException` with the provided error details.

- **Parameters**:
  - `message`: A human-readable description of the error.
  - `exitCode`: The exit code returned by the FFmpeg process.
  - `stdErr`: The standard error output from the FFmpeg process.
  - `commandLine`: The command line used to invoke FFmpeg.
- **Exceptions**: Does not throw exceptions during construction.
