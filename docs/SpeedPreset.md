# SpeedPreset

Fluent builder for configuring an FFmpeg `atempo` filter to change playback speed. It allows specifying the speed factor, whether to preserve pitch, and whether to force reŌĆæencoding, then produces an executable `FFmpegCommand` or its argument list.

## API

### SpeedPreset()
Creates a new, unconfigured instance. No parameters. Returns a readyŌĆætoŌĆæuse builder. Does not throw.

### WithSpeed(double speed)
Sets the playback speed factor.  
- **Parameters**  
  - `speed`: Desired speed multiplier (e.g., `0.5` for half speed, `2.0` for double speed). Must be greater than zero.  
- **Return value**  
  - The same `SpeedPreset` instance to allow chaining.  
- **Exceptions**  
  - `ArgumentOutOfRangeException` if `speed` is less than or equal to zero.

### PreservePitch(bool preserve = true)
Configures whether the audio pitch should be preserved when changing speed.  
- **Parameters**  
  - `preserve`: `true` to keep pitch unchanged, `false` to allow pitch shift with speed. Defaults to `true`.  
- **Return value**  
  - The same `SpeedPreset` instance for further configuration.  
- **Exceptions**  
  - None.

### WithReencode(bool reencode = true)
Determines if the output stream should be reŌĆæencoded rather than copied.  
- **Parameters**  
  - `reencode`: `true` to force reŌĆæencoding, `false` to allow stream copy when possible. Defaults to `true`.  
- **Return value**  
  - The same `SpeedPreset` instance.  
- **Exceptions**  
  - None.

### Build()
Creates an `FFmpegCommand` representing the configured speed change.  
- **Parameters**  
  - None.  
- **Return value**  
  - An `FFmpegCommand` ready for execution or further modification.  
- **Exceptions**  
  - `InvalidOperationException` if `WithSpeed` has not been called or if the speed value leads to an invalid filter chain.

### BuildArguments()
Produces the raw FFmpeg argument list for the current configuration.  
- **Parameters**  
  - None.  
- **Return value**  
  - A `string[]` containing the arguments that would be passed to `ffmpeg.exe`.  
- **Exceptions**  
  - `InvalidOperationException` under the same conditions as `Build()`.

### Run()
Executes the FFmpeg process using the command generated by `Build()`.  
- **Parameters**  
  - None.  
- **Return value**  
  - `void`.  
- **Exceptions**  
  - `FFmpegExecutionException` (or derived) if the FFmpeg process exits with a nonŌĆæzero code or fails to start.  
  - `InvalidOperationException` if the builder is not in a valid state (same as `Build()`).

### ToString()
Returns a humanŌĆæreadable representation of the current configuration, primarily for debugging.  
- **Parameters**  
  - None.  
- **Return value**  
  - A `string` describing the speed, pitch preservation, and reŌĆæencode settings.  
- **Exceptions**  
  - None.

## Usage

```csharp
using FluentFFmpeg;

// Create a SpeedPreset that doubles playback speed while preserving pitch
var preset = new SpeedPreset()
    .WithSpeed(2.0)
    .PreservePitch(true)
    .WithReencode(false);

// Build the command and run it
FFmpegCommand cmd = preset.Build();
cmd.Run(); // throws if ffmpeg fails
```

```csharp
using FluentFFmpeg;

// Obtain the argument list for logging or manual execution
var args = new SpeedPreset()
    .WithSpeed(0.75)
    .PreservePitch(false)
    .WithReencode(true)
    .BuildArguments();

Console.WriteLine(string.Join(" ", args));
// Example output: -filter:a "atempo=0.75,atempo=0.75" -c:a libmp3lame -qscale:a 2
```

## Notes

- The `atempo` filter only accepts speed factors between 0.5 and 2.0 per instance. Values outside this range are automatically chained internally; however, extreme values may require many filters and increase processing time.
- `PreservePitch(true)` inserts the `asetrate` filter to maintain pitch; this only works with audio codecs that support sample rate changes. If the codec does not, FFmpeg will return an error during execution.
- Setting `WithReencode(false)` attempts to copy the stream (`-c copy`). If the selected speed requires filter processing, FFmpeg will ignore the copy request and reŌĆæencode anyway, producing a warning.
- Instances of `SpeedPreset` are mutable and **not** threadŌĆæsafe. Concurrent calls to any of the builder methods from multiple threads may result in undefined state. Once `Build()` is called, the returned `FFmpegCommand` is immutable and safe to use across threads.
- `ToString()` is intended for diagnostics only; its format may change between versions and should not be parsed for programmatic logic.
